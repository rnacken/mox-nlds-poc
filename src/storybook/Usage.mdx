Afhankelijk van je technologie gebruik je het design systeem op andere manieren:

- React? Dan gebruik je de React-componenten
- Iets anders? Dan val je terug op de CSS classes

React componenten (en later componenten voor Vue/Angular/etc) gebruiken zelf onder de motorkap altijd de classes die je zelf ook direct kan gebruiken als je geen React gebruikt.

Via props kan je de styling aanpassen. Een voorbeeld in React:

```tsx
<MoxBox backgroundColor="primary"></MoxBox>
```

Onder de motorkap zit echter een functie die deze property omzet in een className:

```css
<div className="mox-backgroundColor-primary"></div>
```

Op deze manier zijn alle React Atoms opgebouwd en hebben ze een set van dit soort mogelijke props:

```tsx
// Voorbeeld MoxBox:

export const boxStyleProps = [
  "size",
  "inlineSize",
  "blockSize",
  "padding",
  "paddingBlock",
  "paddingInline",
  "paddingBlockStart",
  "paddingBlockEnd",
  "paddingInlineStart",
  "paddingInlineEnd",
  "borderRadius",
  "borderStartStartRadius",
  "borderStartEndRadius",
  "borderEndStartRadius",
  "borderEndEndRadius",
  "borderWidth",
  "borderColor",
  "borderStyle",
  "backgroundColor",
  "gridColumn",
  "gridColumnStart",
  "gridColumnEnd",
] as const satisfies StyleProps;
```

Dit betekent dat een `<MoxBox>` dus deze properties heeft. Via het centrale config-bestand (`mox.config.ts`) is bepaald wat de mogelijke waardes zijn.
Dit is allemaal in typescript opgezet, waardoor je ook type-safety/intellisense hebt op de mogelijke waardes:

<img
  src={TSProps}
  alt="Autocomplete door TS"
  style={{ maxInlineSize: "600px" }}
/>

Sommige properties kunnen voor 1 formaat gekozen worden (zoals hierboven `backgroundColor`). Anderen kunnen daarnaast ook _responsive_ gebruikt worden:

<img
  src={TSPropsResponsive}
  alt="Autocomplete door TS, responsive"
  style={{ maxInlineSize: "700px" }}
/>

Typescript geeft een error als je een waarde gebruikt die niet mogelijk is.

## Componenten gebruiken

Een component kan een **Atom** zijn, of opgebouwd zijn uit meerdere **Atoms** als **Molecule** of (nog complexer) een **Organism**.

Als je componenten uitkiest, is het het handigst om eerst te kijken wat er beschikbaar is in de complexe vorm.

Er is bijvoorbeeld een **organism** voor een `checkbox input` organisme-component dat een checkbox-vakje heeft, een label, een ruimte voor foutmelding, etc.

Als dit te strict is qua opbouw kan je naar een lager level kijken of dit beter past, zoals de **molecule** `checkbox-field`, die enkel het vakje met een label heeft.

Nog meer controle nodig? Kijk dan naar de onafhankelijke **atom** onderdelen, waar je volledige controle hebt over het vakje, het label en de shelf die de `gap` ertussen.

## Componenten aanpassen

De React componenten zoals molecules of organisms zijn opgebouwd uit kleinere blokjes, tot atoms.

Als een molecule of organism niet helemaal naar je wens is, zijn er grofweg 2 opties:

### Tokens

De aanpak voor als je wijziging voldoet aan:

- Je wilt dit _site-wide_
- Het zijn puur visuele aanpassingen
- Je vindt het prima om je tokens-set aan te passen

Pas de tokens aan, zodat deze als `css-variables` door de hele site toegepast worden.

### Bouw je eigen **molecule/organism** component

Dit is ook de aanpak voor als je gewoon een nieuw component wilt opbouwen.

Bouw het met bestaande **atoms** (of **molecules**). Op deze manier kan je de hele structuur van een component anders opzetten.
